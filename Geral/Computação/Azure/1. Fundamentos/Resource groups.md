# Resource Groups

Resource Groups são contêineres lógicos do [[Azure]] usados para organizar recursos.

## 📌 O que é
Um agrupamento que reúne recursos relacionados dentro de uma aplicação ou sistema.

## 📌 Para que serve
- Organizar recursos (API, banco, storage, etc.)
- Gerenciar permissões em grupo
- Facilitar deploy e manutenção

## 📌 Conceito chave
Todo recurso no Azure precisa estar dentro de um Resource Group.

## 📌 Características
- Pode conter vários tipos de recursos
- Não define localização dos recursos, mas possui uma região associada
- Pode ser usado para aplicar políticas e controle de acesso

## 📌 Exemplo
Uma aplicação pode ter:
- API
- Banco de dados
- Storage

Todos dentro do mesmo Resource Group
